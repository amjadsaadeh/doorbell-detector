  <action>
Create data_collection/detector.py as a new, self-contained script. Do NOT modify
data_collector.py. Structure the file in this exact section order:

1. **Module docstring** — describe the script's purpose (pattern-matching doorbell detector,
   Phase 1 skeleton: opens device, loads template, enters capture loop stub).

2. **Imports** — standard library first (argparse, logging, sys, wave), then third-party
   (librosa, numpy as np, pyaudio). paho-mqtt is NOT needed in this phase.

3. **Audio constants** — identical to data_collector.py:
   ```python
   CHANNELS = 1
   FORMAT = pyaudio.paInt16
   RATE = 16000
   ```

4. **Logging setup** — identical format/datefmt to data_collector.py.

5. **Helper: load_template(path: str) -> np.ndarray**
   - Use librosa.load(path, sr=RATE, mono=True) — this resamples automatically if source SR
     differs from 16000. Set sr=RATE so librosa normalizes to float32 in [-1.0, 1.0].
   - Return the numpy array (float32).
   - Do NOT catch exceptions here; let callers handle FileNotFoundError / LibrosaError.

6. **Helper: find_device(p: pyaudio.PyAudio, device_name: str) -> int | None**
   - Iterate p.get_device_count(); match device_name as a substring in info["name"] AND
     info["maxInputChannels"] > 0.
   - Return device index (int) if found, None otherwise.
   - This matches the pattern in data_collector.py but is extracted as a reusable function
     so Phase 2 can call it cleanly.

7. **parse_args() -> argparse.Namespace**
   Arguments to implement:
   - `--template` (str, REQUIRED — use `required=True`): path to the reference doorbell WAV.
     Help: "Path to the reference doorbell WAV file."
   - `--device-name` (str, default "seeed-2mic-voicecard"): audio input device name substring.
     Help: "Audio input device name substring to match (default: seeed-2mic-voicecard)."
   - `--chunk-size-ms` (int, default 500): audio window in milliseconds.
     Help: "Audio capture chunk size in milliseconds (default: 500)."
   Do NOT add threshold, cooldown, MQTT, or save flags — those belong to Phase 2 and 3.

8. **main()**
   Sequence (exit on every error, do NOT continue past a failure):

   a. args = parse_args()

   b. **Template loading** — wrap in try/except:
      ```python
      try:
          template = load_template(args.template)
      except FileNotFoundError:
          log.error("Template file not found: %s", args.template)
          sys.exit(1)
      except Exception as exc:
          log.error("Failed to load template '%s': %s", args.template, exc)
          sys.exit(1)
      ```
      After success: log.info("Loaded template from %s (%d samples)", args.template, len(template))

   c. **Derived constants**:
      ```python
      chunk = int(RATE * args.chunk_size_ms / 1000)
      ```

   d. **PyAudio init + device discovery**:
      ```python
      p = pyaudio.PyAudio()
      device_idx = find_device(p, args.device_name)
      if device_idx is None:
          log.error("Audio device '%s' not found", args.device_name)
          p.terminate()
          sys.exit(1)
      log.info("Using audio device %d: %s", device_idx,
               p.get_device_info_by_index(device_idx)["name"])
      ```

   e. **Stream open**:
      ```python
      stream = p.open(
          format=FORMAT,
          channels=CHANNELS,
          rate=RATE,
          input=True,
          frames_per_buffer=chunk,
          input_device_index=device_idx,
      )
      log.info("Capture loop started — chunk=%d samples (%.0f ms), Ctrl-C to stop",
               chunk, args.chunk_size_ms)
      ```

   f. **Capture loop** (stub for Phase 1 — just reads and discards):
      ```python
      try:
          while True:
              try:
                  _data = stream.read(chunk, exception_on_overflow=False)
              except OSError:
                  log.warning("Buffer overflow — stream read error, continuing")
      except KeyboardInterrupt:
          log.info("Interrupted — shutting down")
      finally:
          stream.stop_stream()
          stream.close()
          p.terminate()
      ```

9. **Entry guard**:
   ```python
   if __name__ == "__main__":
       main()
   ```

Key constraints:

- All log.error() calls go to stderr via the logging framework (correct behavior by default).
- sys.exit(1) for runtime errors (missing file, missing device).
- sys.exit(2) is handled by argparse automatically for missing required args.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → script | User-supplied --template path and --device-name are untrusted filesystem inputs |
| audio device → script | Raw PCM bytes from microphone hardware; overflow possible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | --template CLI arg | accept | Path is user-supplied on own machine; no multi-user exposure in Pi deployment. MQTT password visibility is a known carry-forward constraint (see STATE.md). |
| T-01-02 | Denial of Service | stream.read() buffer overflow | mitigate | Pass exception_on_overflow=False; catch OSError and log warning then continue — matching data_collector.py pattern. |
| T-01-03 | Tampering | Malformed WAV file at --template | mitigate | Wrap librosa.load() in try/except Exception; log.error + sys.exit(1) on any load failure. |
| T-01-04 | Elevation of Privilege | Arbitrary path in --template | accept | Script runs as dedicated service user (Pi deployment); no privilege boundary crossed by reading a file path. |
</threat_model>
