<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| detector.py → MQTT broker | Network connection; broker credentials passed as CLI args |
| MQTT broker → network | TLS optional; insecure mode disables cert verification |
| /etc/doorbell-detector.env → systemd | Env file read as root-owned config; credentials stored as plaintext |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information Disclosure | --mqtt-password in /proc | accept | Known constraint per CLAUDE.md; password visible in /proc/<pid>/cmdline — carry forward as-is |
| T-02-05 | Information Disclosure | MQTT_PASSWORD in /etc/doorbell-detector.env | mitigate | Document in env file comment that /etc/doorbell-detector.env should be chmod 600 owned by root |
| T-02-06 | Tampering | MQTT broker receiving false detections | accept | Cooldown window (default 10s) limits publish rate; no authentication of detection source in v1 |
| T-02-07 | Spoofing | TLS disabled via --mqtt-tls-insecure | accept | Flag is explicitly opt-in and logs a warning; intended for local test brokers only |
| T-02-SC | Tampering | pip installs | accept | No new packages installed in this plan; paho-mqtt is pre-existing in the Pi venv |
</threat_model>
