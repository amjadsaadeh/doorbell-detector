## 1. `src/train_xgboost.py` — full contents

```python
import pandas as pd
import numpy as np
from pathlib import Path
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report
import xgboost as xgb
import yaml
from dvclive.xgb import DVCLiveCallback
from dvclive import Live
from sklearn.preprocessing import LabelEncoder


def prepare_data(df):
    # Convert MFCC features to 1D arrays
    X = np.vstack([x.flatten() for x in df["mfcc_features"]])
    # Convert labels to binary (background=0, non-background=1)
    le = LabelEncoder()
    # Convert to inary problem
    df["label"] = df["label"].apply(
        lambda x: "background" if x == "background" else "bell"
    )
    y = le.fit_transform(df["label"])
    return X, y, le


def main():

    with open("params.yaml", "r") as file:
        params = yaml.safe_load(file)
        
    model_params = params["model"]

    # Load data
    df = pd.read_hdf("./data/balanced_data.h5", key="data")
    X, y, le = prepare_data(df)

    # Split data
    X_train, X_test, y_train, y_test = train_test_split(
        X, y, test_size=params["training"]["test_size"], random_state=42, stratify=y
    )

    with Live() as live:
        # Train model
        model = xgb.XGBClassifier(
            objective="binary:logistic", callbacks=[DVCLiveCallback()], **model_params
        )
        model.fit(X_train, y_train, eval_set=[(X_test, y_test)], verbose=True)

        # Evaluate and save metrics
        y_pred = model.predict(X_test)
        y_pred_decoded = le.inverse_transform(y_pred)
        y_test_decoded = le.inverse_transform(y_test)
        report = classification_report(y_test, y_pred, output_dict=True)

        live.log_sklearn_plot("confusion_matrix", y_test_decoded, y_pred_decoded)
        live.log_metric("f1_score", report["weighted avg"]["f1-score"])
        live.log_metric("recall", report["weighted avg"]["recall"])
        live.log_metric("precision", report["weighted avg"]["precision"])
        # Save model
        model_path = Path("./models/xgboost_model.json")
        model_path.parent.mkdir(exist_ok=True)
        model.save_model(model_path)
        live.log_artifact(model_path)


if __name__ == "__main__":
    main()
```

**dvclive usage specifics:**
- `Live()` context manager wraps the whole train+eval block.
- `DVCLiveCallback()` is passed to `xgb.XGBClassifier(callbacks=[...])` — dvclive auto-logs per-boosting-round eval metrics (train/test logloss etc., from `eval_set`) via this callback, not manually.
- Manually logged metrics: `f1_score`, `recall`, `precision` (all "weighted avg" from `classification_report`, computed on the *encoded* `y_test`/`y_pred`, not the decoded labels — note the confusion matrix plot uses the decoded string labels but the metrics use the encoded/binary ones).
- `live.log_sklearn_plot("confusion_matrix", ...)` — a plot artifact, not a scalar metric.
- `live.log_artifact(model_path)` — logs the saved model file as a dvclive artifact.
- **No explicit `live.log_param(...)` calls** — hyperparameters aren't logged into dvclive directly by this script; they come from `dvc.yaml`'s `params:` stage declaration (`model`, `training` keys in `params.yaml`), which DVC itself tracks via `dvc.lock`/`Live()`'s automatic params capture (dvclive auto-detects `params.yaml` in the working dir).
- Model format: XGBoost's own native serialization via `model.save_model(model_path)` to `models/xgboost_model.json` — plain XGBoost JSON, not pickle/joblib, not an MLflow flavor.
- Hyperparameters used (all from `params.yaml` → `model:` key, splatted into `XGBClassifier(**model_params)`): `n_estimators`, `max_depth`, `learning_rate`, `random_state`. `objective="binary:logistic"` is hardcoded in the script, not in params.yaml. `test_size` comes from `params.yaml` → `training.test_size`.

# Implementation Plan: MLflow Tracking Alongside dvclive

## Design Summary

Add MLflow as a second, best-effort observer inside the same `Live()` block in `src/train_xgboost.py`. Use `mlflow.xgboost.autolog()` (not the blanket `mlflow.autolog()`) for per-round metrics/params/model logging, wrap run creation in a network-failure-tolerant helper so a downed `mlflow.saadeh.dev` never fails `dvc repro`, and gate the whole feature on the presence of `MLFLOW_TRACKING_URI` in the environment so it's fully opt-in and requires zero `dvc.yaml`/`params.yaml` changes.

## 1. `src/train_xgboost.py` — exact diff

```python
import os
import subprocess
import warnings
from contextlib import nullcontext
from pathlib import Path

import mlflow
import mlflow.xgboost
import numpy as np
import pandas as pd
import xgboost as xgb
import yaml
from dvclive import Live
from dvclive.xgb import DVCLiveCallback
from sklearn.metrics import classification_report
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import LabelEncoder

MLFLOW_EXPERIMENT_NAME = "doorbell-detector"


def prepare_data(df):
    X = np.vstack([x.flatten() for x in df["mfcc_features"]])
    le = LabelEncoder()
    df["label"] = df["label"].apply(
        lambda x: "background" if x == "background" else "bell"
    )
    y = le.fit_transform(df["label"])
    return X, y, le


def _git_sha():
    try:
        return (
            subprocess.check_output(["git", "rev-parse", "--short", "HEAD"])
            .decode()
            .strip()
        )
    except Exception:
        return "unknown"


def setup_mlflow():
    """Best-effort MLflow setup. Returns True if the tracking server is
    configured and reachable, False if mlflow tracking should be skipped
    for this run. Never raises — a missing/unreachable MLflow server must
    not fail the dvc repro training stage, only dvclive tracking is
    required for pipeline reproducibility."""
    tracking_uri = os.environ.get("MLFLOW_TRACKING_URI")
    if not tracking_uri:
        return False
    try:
        mlflow.set_tracking_uri(tracking_uri)
        mlflow.set_experiment(MLFLOW_EXPERIMENT_NAME)
        # xgboost-specific autolog only -- NOT the blanket mlflow.autolog(),
        # which would also patch sklearn and autolog train_test_split /
        # LabelEncoder calls above, polluting the run.
        mlflow.xgboost.autolog(log_models=True, log_datasets=False)
        return True
    except Exception as exc:
        warnings.warn(f"MLflow tracking unavailable, continuing with dvclive only: {exc}")
        return False


def start_mlflow_run(enabled):
    if not enabled:
        return nullcontext()
    try:
        run = mlflow.start_run(run_name=f"train-{_git_sha()}")
        dvc_exp_name = os.environ.get("DVC_EXP_NAME")
        if dvc_exp_name:
            mlflow.set_tag("dvc_exp_name", dvc_exp_name)
        mlflow.set_tag("git_sha", _git_sha())
        return run
    except Exception as exc:
        warnings.warn(f"MLflow run creation failed, continuing with dvclive only: {exc}")
        return nullcontext()


def main():
    with open("params.yaml", "r") as file:
        params = yaml.safe_load(file)
    model_params = params["model"]

    df = pd.read_hdf("./data/balanced_data.h5", key="data")
    X, y, le = prepare_data(df)

    X_train, X_test, y_train, y_test = train_test_split(
        X, y, test_size=params["training"]["test_size"], random_state=42, stratify=y
    )

    mlflow_enabled = setup_mlflow()

    with Live() as live, start_mlflow_run(mlflow_enabled):
        if mlflow_enabled:
            try:
                # autolog captures every XGBClassifier constructor kwarg
                # automatically; test_size isn't passed to the estimator,
                # so log it explicitly for parity with dvclive/params.yaml.
                mlflow.log_param("test_size", params["training"]["test_size"])
            except Exception as exc:
                warnings.warn(f"MLflow param logging failed: {exc}")

        model = xgb.XGBClassifier(
            objective="binary:logistic", callbacks=[DVCLiveCallback()], **model_params
        )
        model.fit(X_train, y_train, eval_set=[(X_test, y_test)], verbose=True)

        y_pred = model.predict(X_test)
        y_pred_decoded = le.inverse_transform(y_pred)
        y_test_decoded = le.inverse_transform(y_test)
        report = classification_report(y_test, y_pred, output_dict=True)

        live.log_sklearn_plot("confusion_matrix", y_test_decoded, y_pred_decoded)
        live.log_metric("f1_score", report["weighted avg"]["f1-score"])
        live.log_metric("recall", report["weighted avg"]["recall"])
        live.log_metric("precision", report["weighted avg"]["precision"])
        model_path = Path("./models/xgboost_model.json")
        model_path.parent.mkdir(exist_ok=True)
        model.save_model(model_path)
        live.log_artifact(model_path)

        if mlflow_enabled:
            try:
                mlflow.log_metric("f1_score", report["weighted avg"]["f1-score"])
                mlflow.log_metric("recall", report["weighted avg"]["recall"])
                mlflow.log_metric("precision", report["weighted avg"]["precision"])
            except Exception as exc:
                warnings.warn(f"MLflow post-fit metric logging failed: {exc}")


if __name__ == "__main__":
    main()
```

Notes on design decisions embedded above:

- **Callback compatibility (item 1):** `xgb.XGBClassifier.fit()`'s `callbacks` list is a plain list XGBoost iterates over each boosting round. `mlflow.xgboost.autolog()` patches `fit()` to append its own internal callback to whatever list you pass — it does not replace or inspect `DVCLiveCallback`, so both run independently against the same eval events with no shared state. No conflict.
- **Why not `mlflow.autolog()`:** the blanket autologger also enables `mlflow.sklearn.autolog()`, which would instrument `train_test_split` and `LabelEncoder.fit_transform` above the `Live()`/mlflow-run block and either create spurious autolog side effects or warn about no active run at that point. Scoping to `mlflow.xgboost.autolog()` avoids this entirely.
- **`log_datasets=False`:** dataset lineage is already DVC's job (`balanced_data.h5` is a tracked/hashed output); disabling MLflow's dataset autologging avoids redundant hashing/overhead of the full `X_train`/`X_test` arrays.
- **Model registry (item, scope note 2):** `log_models=True` on autolog already logs a full MLflow Model (pyfunc + native xgboost flavor) as a run artifact — this satisfies "model artifact logging" naturally. The script deliberately does **not** call `mlflow.register_model()` — registering every training run automatically would flood the registry with unpromoted models; promotion to the registry should stay a manual/deliberate step in the MLflow UI, consistent with the requirement that no Pi-deployment/registry-consumption wiring is in scope.
- **Non-fatal design (item 5):** the pre-flight check lives in `setup_mlflow()` (a `set_experiment` call, which contacts the server) and `start_mlflow_run()` (the `start_run` call, a second server round-trip). Both are wrapped in `try/except`, degrading to `contextlib.nullcontext()` so the `with Live() as live, start_mlflow_run(...):` line always succeeds and training proceeds with dvclive alone. Metric/param calls inside the block are also individually wrapped. The one accepted gap: if the server goes down *mid-fit*, autolog's internal per-round callback could itself raise from inside `model.fit()` — this is treated as an acceptable, rare edge case rather than engineered around, to avoid conflating real XGBoost training failures with tracking-infra failures.
