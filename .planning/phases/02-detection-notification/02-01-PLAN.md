## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Microphone input → compute_score() | Raw audio bytes from PyAudio are untrusted (attacker-controlled sound) |
| compute_score() → detection threshold | Score value gates side-effects (log output, later MQTT) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | compute_score() | accept | Pattern-matching is inherently spoofable by replaying the template audio; v1 threat model does not include active adversaries near the microphone |
| T-02-02 | Denial of Service | detection loop | accept | Cooldown (`--cooldown-seconds`, default 10.0) limits detection rate; attacker cannot indefinitely flood logs at arbitrary rate |
| T-02-03 | Tampering | CLI args in /proc | accept | Known constraint per CLAUDE.md; no mitigation planned for v1 |
| T-02-SC | Tampering | pip installs | accept | No new packages installed in this plan; paho-mqtt and all deps are pre-existing in the Pi venv |
</threat_model>
